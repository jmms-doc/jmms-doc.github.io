0.4.x

0.4.7-2

  • 修复jmms run在windows7/8出现Could use "nc"...的告警信息
  • 修复jmms run首次运行下载依赖包过慢导致开发人员以为卡死的问题
  • 修复jmms run在Windows下CTRL+C终止报错的问题

0.4.7

  • 修复在{Entity}.json中定义嵌套attrs配置属性报错的问题
  • 修复在{Entity}.json中定义className后引用报错的问题

0.4.6

  • 修复js中全局变量会优先于请求参数被获取到的问题
  • 修复js操作中返回的数组不能被正确序列化的问题

0.4.5

  • 修复路径/../{id}/..在打包运行模式下会被url编码的问题

0.4.4

  • 更新leap版本为0.7.4b
  • 修复开启OAuth2后CORS请求不能正确处理的问题

0.4.3

  • 更新leap版本为0.7.3b
  • 少量改进文档生成模板
  • 改进基于实体的数据权限
  • 支持基于SQL的数据权限
  • 自动把选项集转换为Swagger的enum

0.4.2

  • 支持通过{Entity}_ex.json来扩展一个已有的实体
  • 修复ID属性不能被创建的问题
  • 在API级别增加upgradeSchema配置来设置是否自动更新数据库模型
  • 默认关闭实体删除操作的cascade_delete功能，另外增加配置进行打开

0.4.1

  • 更新leap版本为0.7.2b
  • 修复文档生成少量bugs
  • 支持internal实体
  • 给实体增加filters配置实现数据权限
  • 在Javascript中支持env变量
  • 修复单元测试插件中不能识别正常OAuth2请求的问题
